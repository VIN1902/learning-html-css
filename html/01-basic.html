Use index.html as file name if you want that to be recognized as entry point by different s/w, also its the standard.

Emmet is s/w that provides functionality to write fast html and css. vscode has it pre-configured. provides abbreviation for boilerplate too.

Read mdn documentation for filling gaps in knowledge.

HTML is a structure language before hypertext or anything. The point is to provide structure to web page. Others like markdown do that too but HTML is most supported universally. Everthing has a browser by default thesedays.

Heading tags are not just font-sizes, they provide relevance to page structure by assigning importance to content, not only to reader but to search-engines.

Tag is the smallest unit, Element is opening tag (along with attribute plus its value) + content + closing tag.

For tags its recommended that you provide some closing, either proper closing tag or self-closing. If not then browser won't give errors but certain frameworks like React might.


HTML is not case-sensitive, no need to write tag capital.

Nested element

Elements can be nested with other elements inside them. like using span tag in heading above.

'#' value to attributes like href in anchor tag means 'stay at current page and scroll to the top'. '#' is fragment identifier pointing to top of current page. Use it as

  • A 'scroll to top' link. Click here to go at top
  • Just a temporary placeholder for actual links cause it will either reload or scroll to top of page, which is often unwanted.
  • To jump to any element on current page by specifying its id. ex: href = '#someId'

Some attributes are required for a tag others are optional. like title attribute is optional, it provides a small section when hovered over an element.
