Global attributes: Class and Id

The purpose/point of classes and ids is to target and select elements. Styling after selection is only one usecase.

Targeting elements is usefull in many frameworks for styling or for other tasks.

In JS the convention is to use Ids for targeting only unique elements and if more unique elements show-up then produce more ids. This way page only has few ids and lots of classes.

But in CSS convention almost all frameworks produce only classes.

In self-written code best practice is to use only one id per web-page and rest classes. But frameworks work with different approach and they programaticaly generate ids & classes instead of hard-writing them.
