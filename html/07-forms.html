Forms

Attributes

  • action: tells browser where to send data after submit button is clicked.
    • usually a url is passed as value, to indicate the server-endpoint.
    • if attribute is omitted then data is sent to current page.
  • method: specifies how the form data is sent to server.
    • GET: send data as part of url (?key1=value1&key2=value2). Data is visible and Not secure.
    • POST: send data inside request body. Data is not visible and secure.
  • enctype: how to encode/format the data before sending. Applicable only with post method.
    • application/x-www-form-urlencoded: url-encoded, default, for simple text data.
    • multipart/form-data: non-encoded, for file uploads.
    • text/plain: non-encoded, for plain-text, rare.
  • value: this is attribute for input fields. specify default value of input field that'll be sent.
    • in button-like input tags value act as their name.
  • placholder: its not the value but an example. no data will be sent with just placeholder.

Practices

  • Give a label tag with any input tag to identify the corresponding input field by clicking on input title.
    • Connect both by specifying same name used in id attribute of input tag, inside for attribute of label tag.
  • Styling is done via CSS.
  • Things like submit are a button or input field doesn't matter much.
    • With button tag you can imbed any other html like img etc. whereas with input tag only value can be set.
