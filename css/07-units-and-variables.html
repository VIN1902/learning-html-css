Units and Variables in CSS

CSS Units

1. Absolute Units
They specifiy a fixed size. Use pixels (px) for web.
Px is a measurment of length that is relative to the expected viewing distance of device being used.
Simply 1px is the size that is necessary for any device to make something visible.
Use px sparingly (for borders, fine-tuning UI).
2. Relative Units
They depend upon and are realtive to size of fonts,viewport,etc. Use em, rem, ch, vh/vw, % for web.
Need: Every browser has option to change font size (set to 16px in most).
If a user changes this for accesibility reasons then their preference gets overriden by you, if absolute unit like px is used for something like font-size.
2.1 em Unit
Relative to font size of its parent.
If the parent doesn't define one, it eventually falls back to the document's root font size (16px in most browsers).
Use case: You might set a container's font size to 1em and its child element's to 2em to make it larger relative to the parent.
Issue: In nested elements, em can become confusing because its size compounds through the chain of parent elements.
When em is used for properties other than font-size (like padding or margin), it's relative to the font size of the element itself, not its parent.
Tip: Use em for spacing (padding, margin) and rem for font sizes.
2.2 rem Unit
Always relative to the root document's font size (typically 16px).
If you use 2rem anywhere, it always means 2 x 16px, regardless of nesting.
        
            .button {
                background-color: red;
                color: white;
                font-size: 1.5rem;
                padding: 1em;
            }
2.3 % Unit
Relative to the size of the parent element.
It's commonly used for widths, heights, and positioning to make layouts responsive.
2.4 vh or vw Unit
Relative to viewport height or width. 1vh means 1% of viewport height.
2.5 ch Unit
Measures the width of the character “0” (zero) in the current font.
Recommended line lengths for optimal reading typically fall between 45 and 75 characters.
Using ch units for max-width on text containers (like paragraphs) helps ensure consistent line lengths, even when the font size changes.
        
            p {
                max-width: 65ch; /* Limits paragraph width to approximately 65 "0" characters */
            }

            input[type="number"] {
                width: 4ch; /* Useful for fixed-width inputs like a year field */
            }

CSS Variables

Store values and reuse them by refering to variable name, instead of manually using exact value everywhere.

Makes its much more managable and scalable.

The :root selector is commonly used to define global variables, making them accessible throughout the entire document. You can also define variables within specific selectors to scope them locally.

        
            :root {
                --primary-color: #3498db; /* Define a global variable for a primary blue color */
                --secondary-color: #2ecc71; /* Define a global variable for a secondary green color */
                --font-size-base: 16px; /* Define a global variable for a base font size */
            }

            body {
              background-color: var(--primary-color); /* Use the primary color for the body background */
              font-family: Arial, sans-serif;
              font-size: var(--font-size-base); /* Use the base font size */
            }

            h1 {
              color: var(--secondary-color); /* Use the secondary color for heading text */
              font-size: calc(var(--font-size-base) * 1.5); /* Calculate heading font size based on base */
            }

            button {
              background-color: var(--primary-color);
              color: white;
              padding: 10px 20px;
              border: none;
              border-radius: 5px;
            }
